Minesweeper AI

Description:

If you are like me, you’ve spent too many hours playing the Google Chrome minigames trying to beat a high score. So I made Stephen, the Google Minesweeper AI to play for me. The program takes control of your mouse and keyboard and self navigates to the game and plays by itself. Do you have any Minesweeper time trial records? Stephen does—36 seconds on the medium level. The code was written to work on Raspbian with the Raspberry Pi.

Technical Bits:

This program was coded in Python using the PyAutoGUI pip library. Once the game begins the program moves the cursor out of the way, takes a screenshot of the board, and using some rudimentary library-free image processing interprets the colors and numbers and spits out an array that represents the current game state. From this array, a series of logic rules are applied, and the program plans out all of the possible moves it can make, mapping them to a series of coordinates on the screen. This list of coordinates then gets executed, the cursor automatically does it’s thing, and milliseconds later it’s back off to the side taking the next screenshot.

Challenges:

This was one of my first real programs in Python, and my first time ever trusting my code with control over my mouse and keyboard (spoiler alert, I regretted it fast). Mistake #1: I didn’t code a fail safe, and when my sketchy code found its way clicking into my system preferences I learned my lesson. Another problem came up when the design of the game changed ever so slightly halfway through the development process where the rgb values of the numbers on the board all shifted a bit, which caused the image processing code to no longer return accurate readings of the current game state.
