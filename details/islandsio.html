Islands.io

Description:

A unique cross between a base defense, resource collection game, and a classic, fast gameplay .io game. Play this fun game for 2-4 players with your friends in private rooms that let you enjoy all the benefits of multiplayer gaming for free in a completely online, download free environment. Islands.io has been in development for the last 6 months, and has just now entered the beta testing phase. It may be down for development, but if you would like to play the game with friends now, just go to Islandsio.com

Technical Bits:

This game was coded with HTML5, CSS, and JavaScript. The server-side code is written using the socket.io library version 2.3.0, along with express, node.js, and the yarn package manager. There is not a single image file in the entire game, instead every single word, graphic, player, tile, and bullet is being drawn using vanilla JavaScript drawing operations in a full-screen canvas. Although custom sprites or drawing libraries like p5.js might have been easier, I decided to challenge myself in order to get a better understanding of how to navigate the JavaScript canvas.

Challenges:

This being my first time creating a multiplayer application, I didn’t exactly design my first iteration with efficiency in mind—which became abundantly clear when the game ran at 3 fps. To get to the smooth 30 fps we’re at today, I had to do some major refactoring, primarily uprooting the way in which data was sent back and forth so that now, only changes to player and bullet position along with tile updates are sent to the server and back out to the clients. This included adding optimizations like render distance, more efficient array sorting methods, and moving all actual game logic to the server, leaving the client to simply get the data and render it on screen.
