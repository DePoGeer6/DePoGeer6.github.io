Dot Packing

Description:

Growing up colorblind, I loathed the Ishihara Dot Tests. What started as an experiment in programmatic dot packing algorithms turned into a crossover with image processing and random color generation. Upload any black and white photo, and you can turn it into an Ishihara test, or upload a full color photo and turn it into an array of colorful dots that try to regulate themselves in size to create a detailed picture. By toggling a handful of parameters such as maximum dot count, minimum and maximum dot size, and limiting dot growth rate, you can really control the level of detail in your results.

Technical Bits:

This program was built in the Processing IDE using Java. First an image is uploaded, and then some simple image processing code is run that creates an arraylist of potential circle locations based on the rgb values of the image. Every frame, a new set of circles is added in open positions, and the existing circles grow by a few pixels until there are no available spots left for a new circle to spawn with the minimum defined radius. In the image rendering mode, the circles inherit the color of the pixel in the same location on the uploaded photo, hence the colorized image.

Challenges:

Java Processing is one of my most comfortable visual coding environments, so programming wise, there were very few issues that came up. Logically, I struggled with figuring out how to constrain the generated circles into a certain shape (i.e the circular ishihara tests) in a modular way without having to upload already circular images. At first my solution was to generate a full rectangular image worth of circles then simply crop it, but it left a lot of half cut off circles around the border.
